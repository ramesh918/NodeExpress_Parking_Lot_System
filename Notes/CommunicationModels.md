
| **Model**             | **Description** | **Characteristics** | **Examples** | **Real-Time Use Cases** |
|----------------------|---------------|---------------------|-------------|----------------------|
| **Request-Response** | Client sends a request, and the server processes and sends back a response. | - Synchronous communication <br> - Client waits for a response <br> - Common in web apps & APIs | HTTP requests, REST APIs, GraphQL | Browsing websites (Google Search, Amazon shopping), Online banking, Food delivery apps (Swiggy, Uber Eats) |
| **Publish-Subscribe (Pub-Sub)** | A publisher sends messages to a broker, which distributes them to multiple subscribers. | - Asynchronous <br> - Event-driven architecture <br> - Decoupled communication | MQTT, Apache Kafka, Redis Pub/Sub | Stock market price updates, Real-time chat notifications (Slack, WhatsApp), IoT applications (smart home automation) |
| **Push Model** | Server actively pushes updates to the client without waiting for a request. | - Real-time updates <br> - Reduces client polling overhead <br> - Efficient for notifications | WebSockets, Firebase Cloud Messaging (FCM), Server-Sent Events (SSE) | Mobile push notifications (WhatsApp, Instagram), Live sports scores, News updates (BBC, CNN) |
| **Polling Model** | The client periodically checks the server for updates. | - Simple to implement <br> - Less efficient <br> - Causes unnecessary network traffic | AJAX polling, Checking email updates | Checking for new emails (Gmail, Outlook), Refreshing social media feeds (Facebook, Twitter) |
| **Streaming Model** | Continuous flow of data from the server to the client in real-time. | - Low latency <br> - Used for media streaming & real-time data <br> - Data is received in chunks | YouTube Live, Netflix, HTTP Live Streaming (HLS), RTSP | Watching live sports (ESPN, Hotstar), Online gaming (Twitch, Discord voice chat), Video conferencing (Zoom, Google Meet) |
| **Peer-to-Peer (P2P)** | Clients communicate directly without a central server. | - Decentralized <br> - Efficient for large-scale systems <br> - Can be difficult to manage security | BitTorrent, Blockchain, Skype (older versions) | File sharing (Torrent downloads), Cryptocurrencies (Bitcoin, Ethereum), Decentralized messaging apps (Session, Briar) |
